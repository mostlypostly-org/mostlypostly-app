| Property | Value |

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), badge inner padding (`px-2 py-0.5`) |
| sm | 8px | Compact element spacing (`gap-2`), button inner padding (`px-2 py-1`) |
| md | 16px | Default element spacing (`gap-4`), card body padding (`px-4`) |
| lg | 24px | Section padding (`px-6 py-4`), card outer spacing (`mb-6`) |
| xl | 32px | Layout gaps between major blocks |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions: Touch targets for row-level action buttons use minimum height of `py-1` (8px vertical) + `px-2` (8px horizontal) = 36px minimum tap area — acceptable for desktop-primary admin UI. The recycle banner uses `px-5 py-3` (20px / 12px) consistent with `failedBanner` pattern in manager.js.

Source: Codebase scan of admin.js, manager.js, dashboard.js established patterns

## Color

All tokens are defined in CLAUDE.md Brand Palette. Hex values are canonical.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#F8FAFC` (`mpBg`) | Page background, empty cells, placeholder divs |
| Secondary (30%) | `#FFFFFF` (`mpCard`) + `#E2E8F0` (`mpBorder`) | Card backgrounds, table rows, dividers, input borders |
| Accent (10%) | `#3B72B9` (`mpAccent`) | See reserved list below |
| Accent Light | `#EBF3FF` (`mpAccentLight`) | Accent button hover background, active nav state, info badge fill |
| Destructive | `#DC2626` (`red-600`) | Block toggle active state, destructive hover states |
| Text Primary | `#2B2D35` (`mpCharcoal`) | All body text, headings |
| Text Secondary | `#7A7C85` (`mpMuted`) | Helper text, secondary labels, empty states |

**Accent (`#3B72B9`) reserved for:**
- Active sidebar nav item background + text
- Recycle row-level button (border + text, hover fills to solid)
- Auto-recycle notice banner text and link underline
- Admin tab active indicator (border-bottom on active tab)
- Focus ring on Admin select dropdowns (`focus:ring-mpAccent`)
- "Coming Up" status badge text (`text-blue-600` / `bg-blue-50` — closest Tailwind equivalent to accent light)

**Semantic colors for this phase:**
- `bg-blue-50 border-blue-200 text-blue-700` — auto-recycle notice banner (info, non-blocking)
- `bg-red-100 text-red-600 border-red-300` — Block toggle when active (post is blocked from recycling)
- `bg-mpBg text-mpMuted border-mpBorder hover:border-red-300 hover:text-red-500` — Block toggle when inactive (default, destructive-on-hover affordance)
- `bg-green-50 border-green-200 text-green-800` — Admin settings saved confirmation notice

Source: CLAUDE.md Brand Palette, manager.js failedBanner pattern, admin.js notice pattern, RESEARCH.md Pattern 4 and Pattern 5

## Component Inventory

The following net-new UI components are introduced in this phase. Each follows an established codebase pattern.

### 1. Auto-Recycle Notice Banner (manager.js dashboard)

- **Location:** Top of manager dashboard, above `failedBanner` (or stacked below it if both active)
- **Trigger:** Renders when `recycledThisWeek > 0`
- **Pattern:** Mirrors `failedBanner` — `rounded-xl border bg-blue-50 border-blue-200 px-5 py-3 mb-6`
- **Contents:** "X post(s) were auto-recycled this week." + link "View in Database" → `/dashboard?salon={id}&status=published` + dismiss button (`&times;` character, JS DOM removal)
- **Dismiss:** Client-side JS only (`document.getElementById('recycle-notice').remove()`) — no persistence, reappears on next page load
- **Non-blocking:** Does not prevent any existing dashboard action

### 2. Recycle Row Button (dashboard.js published post rows)

- **Location:** Inline on each row where `status = 'published'`, in an "Actions" column
- **Default state:** `bg-mpAccentLight text-mpAccent border-mpAccent` — accent fill, accent text
- **Hover state:** `hover:bg-mpAccent hover:text-white` — solid fill
- **Size:** `px-2 py-1 text-[10px] font-medium rounded`
- **Form:** `<form method="POST" action="/dashboard/recycle-post">` with hidden `post_id` and `salon` inputs
- **Non-published rows:** Render `—` in the Actions cell (no button)

### 3. Block Toggle Button (dashboard.js published post rows)

- **Location:** Inline alongside Recycle button, same Actions cell, `flex gap-2` container
- **Inactive state (not blocked):** `bg-mpBg text-mpMuted border-mpBorder hover:border-red-300 hover:text-red-500`
- **Active state (blocked):** `bg-red-100 text-red-600 border-red-300`
- **Label:** "Block" (inactive) / "Blocked" (active)
- **Size:** `px-2 py-1 text-[10px] font-medium rounded`
- **Form:** `<form method="POST" action="/dashboard/toggle-block">` with hidden `post_id` and `salon` inputs
- **No confirmation dialog** — toggle is reversible; no destructive consequence
- **Note:** `block_from_recycle` and `recycled_from_id` must be added to the dashboard.js SELECT query or the button state cannot render correctly (RESEARCH.md Pitfall 5)

### 4. Auto-Recycle Admin Toggle (admin.js — Posting tab)

- **Location:** Admin → Posting tab, in the Manager Rules section, below existing `require_manager_approval` toggle
- **Control:** `<select>` using existing `selectCls` pattern: `w-full rounded-lg border border-gray-200 bg-gray-50 px-3 py-2 text-sm text-mpCharcoal focus:border-mpAccent focus:outline-none focus:ring-1 focus:ring-mpAccent`
- **Field name:** `auto_recycle`
- **Options:** "Disabled" (value 0) / "Enabled" (value 1)
- **Visible to:** All plans (Starter, Growth, Pro, trial)
- **Label:** "Auto-Recycle" with helper text: "Automatically requeue top-performing posts when your queue runs low."

### 5. Caption Refresh Admin Toggle (admin.js — Posting tab)

- **Location:** Below `auto_recycle` toggle in Manager Rules section
- **Visibility gate:** Only rendered for Growth and Pro plan salons. For Starter/trial: hidden entirely (no opacity or lock — simply not in the DOM). This is a plan upsell differentiator.
- **Control:** Same `selectCls` select pattern
- **Field name:** `caption_refresh_on_recycle`
- **Options:** "Disabled — use original caption" (value 0) / "Enabled — refresh via AI" (value 1)
- **Default:** Disabled (value 0)
- **Label:** "Caption Refresh on Recycle" with helper text: "When enabled, recycled posts get a fresh AI-written caption. Growth and Pro plans only."

Source: CONTEXT.md Decisions, RESEARCH.md Pattern 3 and Pattern 4

## Interaction States

### Recycle Button — POST submit

1. Manager clicks "Recycle" on a published post row
2. Form POSTs to `/dashboard/recycle-post`
3. Server clones the post, calls `enqueuePost()`, redirects back to database view with `?notice=Post+recycled`
4. Database view renders green success notice: `rounded-xl border border-green-200 bg-green-50 px-4 py-3 mb-4 text-xs text-green-800 font-medium`
5. The new recycled post appears in the queue with `recycled_from_id` set

### Block Toggle — POST submit

1. Manager clicks "Block" or "Blocked" button on a published post row
2. Form POSTs to `/dashboard/toggle-block`
3. Server flips `block_from_recycle` value, redirects back to same database view URL
4. Button re-renders in correct active/inactive state on page reload
5. No toast or banner needed — state is visually obvious from button appearance

### Auto-Recycle Notice — dismiss

1. Manager clicks `&times;` button on recycle notice
2. JS removes `#recycle-notice` element from DOM
3. No server call, no cookie — notice reappears on next full page load

### Admin Toggles — save

1. Manager changes `auto_recycle` or `caption_refresh_on_recycle` select
2. Form POSTs to existing `update-manager-rules` endpoint in admin.js
3. Redirect back to Admin → Posting tab with `?notice=Settings+saved`
4. Green notice banner renders at top of admin page

Source: CONTEXT.md Decisions (undo = delete recycled copy), RESEARCH.md Pattern 3, 4, 5, codebase scan

## Plan-Gated UI Rules

| UI Element | Starter/Trial | Growth | Pro |
|------------|--------------|--------|-----|
| Auto-Recycle toggle (admin) | Visible, configurable | Visible, configurable | Visible, configurable |
| Caption Refresh toggle (admin) | Hidden entirely | Visible, configurable | Visible, configurable |
| Recycle button (database view) | Visible | Visible | Visible |
| Block toggle (database view) | Visible | Visible | Visible |
| Auto-recycle notice banner | Visible | Visible | Visible |

The caption refresh toggle is the only plan-gated UI element. It is hidden for Starter/trial — not greyed out, not locked with an icon, simply not rendered. This matches the project convention: plan-gated features are either rendered or not, not shown in a disabled state (avoids confusion).

Source: CONTEXT.md Decisions (Caption Refresh on Recycle section), RESEARCH.md Pattern 3
