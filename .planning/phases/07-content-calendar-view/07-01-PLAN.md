**`calendarPillClass(post)` function** -- returns Tailwind class string:
- If `post.vendor_campaign_id` is truthy: return `"bg-purple-100 text-purple-700 border-purple-200"`
- Else use a map keyed by `post.post_type`:
  - `standard_post`: `"bg-blue-100 text-blue-700"`
  - `before_after` and `before_after_post`: `"bg-teal-100 text-teal-700"`
  - `availability`: `"bg-green-100 text-green-700"`
  - `promotion` and `promotions`: `"bg-amber-100 text-amber-700"`
  - `celebration` and `celebration_story`: `"bg-pink-100 text-pink-700"`
  - `reel`: `"bg-indigo-100 text-indigo-700"`
- If `post.status === "failed"`: return `"bg-red-100 text-red-700"` (overrides type)
- Default fallback: `"bg-gray-100 text-gray-600"`

**`calendarPillLabel(post)` function** -- returns display label:
- If `post.vendor_campaign_id` and `post.vendor_name`: return the vendor_name (e.g. "Aveda")
- Else use a map: `standard_post`->"Post", `before_after`/`before_after_post`->"B/A", `availability`->"Avail", `promotion`/`promotions`->"Promo", `celebration`/`celebration_story`->"Celeb", `reel`->"Reel"
- If `post.status === "failed"`: return "Failed"
- Default: post.post_type or "Post"

**GET `/` handler** (the main calendar page):

1. Read `salon_id` from `req.session.salon_id`. Read salon timezone: `db.prepare("SELECT timezone FROM salons WHERE slug = ?").get(salon_id)`. Default timezone to `"America/Indiana/Indianapolis"` if not set.

2. Read `?month=YYYY-MM` from query string. If not provided or invalid, default to current month in salon timezone: `DateTime.now().setZone(tz).toFormat("yyyy-LL")`.

3. Compute 4-week grid boundaries:
   - `monthStart = DateTime.fromISO(month + "-01", { zone: tz }).startOf("month")`
   - `gridStart = monthStart.startOf("week")` (Luxon weeks start Monday by default -- use `.startOf("week", { useLocaleWeeks: false })` or adjust to Sunday start: `monthStart.startOf("week").minus({ days: monthStart.startOf("week").weekday === 7 ? 0 : monthStart.weekday })` -- actually simplest: compute the Sunday at or before monthStart: `let gridStart = monthStart; while (gridStart.weekday !== 7) gridStart = gridStart.minus({ days: 1 });`
   - `gridEnd = gridStart.plus({ weeks: 5 }).endOf("day")` (show 5 weeks to cover any month layout)
   - Convert to UTC for DB query: `rangeStartUtc = gridStart.startOf("day").toUTC().toFormat("yyyy-LL-dd HH:mm:ss")`, `rangeEndUtc = gridEnd.endOf("day").toUTC().toFormat("yyyy-LL-dd HH:mm:ss")`

4. Run the locked DB query:
```sql
SELECT p.id, p.post_type, p.status, p.scheduled_for, p.published_at, p.stylist_name,
       p.image_url, p.final_caption, p.vendor_campaign_id, vc.vendor_name
FROM posts p
LEFT JOIN vendor_campaigns vc ON p.vendor_campaign_id = vc.id
WHERE p.salon_id = ?
  AND p.status NOT IN ('draft', 'cancelled')
  AND (p.scheduled_for BETWEEN ? AND ? OR p.published_at BETWEEN ? AND ?)
ORDER BY COALESCE(p.scheduled_for, p.published_at) ASC
```
Parameters: `[salon_id, rangeStartUtc, rangeEndUtc, rangeStartUtc, rangeEndUtc]`

5. Group posts by local date: for each post, compute `localDate = DateTime.fromSQL(post.scheduled_for || post.published_at, { zone: "utc" }).setZone(tz).toFormat("yyyy-LL-dd")`. Build a `Map<string, post[]>` keyed by localDate.

6. Build the HTML grid:
   - Month header: `<` prev month link, month name + year (e.g. "March 2026"), `>` next month link. Prev/next use `?month=YYYY-MM` query param.
   - Day-of-week header row: Sun, Mon, Tue, Wed, Thu, Fri, Sat
   - 5 rows of 7 day cells. Each cell:
     - `data-date="YYYY-MM-DD"` attribute (for drag target and day panel click)
     - Day number. Grey out days outside the current month (`text-gray-300`).
     - Today highlighted with `ring-2 ring-mpAccent` border.
     - For each post on that day: render a small pill `<div class="calendar-post-card cursor-grab px-1.5 py-0.5 rounded text-[10px] font-semibold truncate mb-0.5 ${calendarPillClass(post)}" data-id="${post.id}">${safe(calendarPillLabel(post))}</div>`
     - If more than 3 posts, show first 3 + a "+N more" indicator.
     - Cell is clickable (onclick opens day panel -- wired in client JS).
   - Each cell has CSS class `calendar-day-cell` and `min-h-[100px]` for consistent height.

7. Client-side JS (inline `<script>` at bottom of body):
   - Day cell click handler: `document.querySelectorAll('.calendar-day-cell').forEach(cell => cell.addEventListener('click', (e) => { if (e.target.closest('.calendar-post-card')) return; openDayPanel(cell.dataset.date); }));`
   - `openDayPanel(date)` function: fetch `/manager/calendar/day/${date}`, set innerHTML of `#day-panel-content`, show `#day-panel` (remove `hidden` class, add slide-in).
   - Close panel button handler.
   - SortableJS CDN script tag: `<script src="https://cdn.jsdelivr.net/npm/sortablejs@1.15.2/Sortable.min.js"></script>` (same as postQueue.js uses).
   - SortableJS initialization on each `.calendar-day-cell` with `group: { name: 'calendar-posts', pull: true, put: true }`, `draggable: '.calendar-post-card'`, `animation: 150`, `ghostClass: 'opacity-40'`. The `onEnd` handler is a stub in this plan (Plan 02 wires it to the reschedule endpoint).

8. Day panel container (right side slide-out):
   - `<div id="day-panel" class="hidden fixed inset-y-0 right-0 z-40 w-96 max-w-full bg-white border-l border-mpBorder shadow-2xl overflow-y-auto transition-transform">`
   - Close button (X) at top right.
   - `<div id="day-panel-content"></div>` where the fragment loads.
   - Overlay backdrop: `<div id="day-panel-backdrop" class="hidden fixed inset-0 z-30 bg-black/20" onclick="closeDayPanel()"></div>`

9. Wrap everything in `pageShell({ title: "Calendar", body, current: "calendar", salon_id, manager_id: req.session.manager_id })`.
