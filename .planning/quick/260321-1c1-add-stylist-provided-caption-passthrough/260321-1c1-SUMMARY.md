# Quick Task 260321-1c1: Stylist-Provided Caption Passthrough Summary

**One-liner:** Pure heuristic `isRealCaption()` detects real captions (8+ words or emoji+5words), skips AI generation and keeps text verbatim; stylist portal shows prominent blue "Generate AI Version" button when caption was kept.

## What Was Built

### `src/core/captionDetection.js` (new)
Pure exported function `isRealCaption(text)`:
- **8+ words + secondary signal** (capital start, punctuation, or emoji) → true (keep verbatim)
- **Emoji + 5+ words + secondary signal** → true (catches concise real captions like "This turned out so beautiful! Love the dimension ✨")
- Everything else → false (AI generates as before)
- Zero dependencies, runs in <1ms, 10/10 test cases pass

### `src/core/messageRouter.js` (modified)
Two integration points:

**`processNewImageFlow`:**
- Checks `captionKept = postType === "standard_post" && isRealCaption(text)` after classification
- If true: skips `generateCaption()`, builds `aiJson = { caption: text, hashtags: [], cta: "" }`
- Portal URL gets `&caption_kept=1` appended
- SMS: "Your caption was kept! Review and approve here: {url}..."
- If false: existing AI flow unchanged

**`createCoordinatorPost`:**
- Same heuristic check on `messageBody`
- If kept: caption set to `messageBody.trim()`, no `generateCaption()` call
- Portal URL gets `&caption_kept=1`
- SMS: "Got it! Your caption for {name} was kept. Review here: {url}..."
- If not kept: existing AI flow unchanged

### `src/routes/stylistPortal.js` (modified)
GET `/:id` now reads `captionKept = req.query.caption_kept === "1"`:

**When captionKept is true (new UI):**
```
[Image]
"Generate AI Version" (blue button, full width, prominent above caption)
"Add direction for AI (optional)" textarea
"Caption from your message" label (blue, small)
[Caption box — their kept text]
[Service type chips]
[Add hashtags]
[Submit for Manager Review]
```

**When captionKept is false (unchanged AI flow):**
```
[Image]
[Caption Preview box]
[Add your input (optional)] textarea
[Regenerate with AI] (subtle button)
[Service type chips]
[Add hashtags]
[Submit for Manager Review]
```

"Generate AI Version" submits to the existing `POST /:id/regenerate` endpoint. After redirect (`?regen=1`), `caption_kept` param is absent — the caption is now AI-generated and the normal view renders.
