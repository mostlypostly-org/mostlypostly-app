# Quick Task 260321-18y: Fix Platform Console Apply All Buttons Summary

**One-liner:** Fixed Apply All toggles in Platform Console routing table by replacing inline `onchange` arrow functions with `data-apply-all` attribute + document-level event delegation.

## What Was Done

The "Apply All" toggle row in the Platform Console Standard Routing Defaults table had a broken inline `onchange` handler:

```js
onchange="document.querySelectorAll('.col-${plat}').forEach(cb => { cb.checked = this.checked; })"
```

While `this` in an inline handler correctly refers to the element, CSP restrictions from the previous fix (260320-ate) may have been blocking inline handlers. The defensive approach matches the established pattern in this file: remove inline handlers entirely and use event delegation.

**Fix applied:**
1. Removed the `onchange` attribute from each Apply All checkbox
2. Added `data-apply-all="${plat}"` attribute (facebook/instagram/gmb/tiktok) to identify the platform
3. Added a single `document.addEventListener('change', ...)` delegation handler in the existing `<script>` block that reads `e.target.closest('[data-apply-all]')`, extracts the platform name, and sets all `.col-{plat}` checkboxes to match

## Deviations from Plan

None — plan executed exactly as written. The defensive event-delegation approach described in the plan was implemented.
